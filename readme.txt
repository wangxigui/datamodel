数据模型

OLTP: 关系型数据库，数据库的增删改查，底层数据更新，银行／证券

OLAP: 面向决策层的数据建模分析

1. ER 实体关系模型
    1. 复杂，非业务人员很难理解, 随便一个业务7-8张表
    2. 3NF， 3范式建模，一致性，数据精简无冗余

2. 数据层次框架： 操作数据层(格式化的原始数据)， 公共数据层(加工后的公共通用数据)，接口层，应用数据层

3. Kimball 维度建模
    1. 灵活性较差，数据冗余
    3. 大宽表，关系模型的层次结构展开平铺而成

4. 事实表+维度表， 维度表使用反范式建模
